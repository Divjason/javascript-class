1."동기"란 무엇일까?
- 여러개의 작업을 순서대로, 하나씩 처리하는 방식
- 작업을 실행하고 처리하는 역할을 하는 존재를 Thread라고 부름
- JavaScript는 "동기"적으로 코드를 실행한다
- 동기 방식에는 치명적인 단점이 존재한다
- JavaScript 엔진에는 Thread가 1개 밖에 없음 = Single Thread

2."비동기"란 무엇일까?
- 동기적이지 않다는 뜻 = 작업을 순서대로 처리하지 않음

3.JavaScript에서 "비동기"적으로 처리하게 해주는 함수
- setTimeout( )

4.JavaScript에서 "비동기" 작업들의 처리 방법
- JavaScript 엔진이 아닌 Web APIs에서 실행 됨
- Web APIs : 브라우저가 직접 관리하는 별도의 공간

5.Promise란?
- 비동기 작업을 효율적으로 처리할 수 있도록 도와주는 JavaScript의 내장 객체
- Promise는 비동기 작업을 감싸는 객체
> 비동기 작업 실행
> 비동기 작업 상태 관리
> 비동기 작업 결과 저장

6.Promise의 3가지 상태
- 대기 (Pending) : 아직 작업이 완료되지 않은 상태
- 대기 상태에서 성공 상태로 전환되는 것 : resolve 상태 (*해결)
- 성공 (Fulfilled) : 비동기 작업이 성공적으로 마무리 된 상태
- 대기 상태에서 실패 상태로 전환되는 것 : reject 상태 (*거부)
- 실패 (Rejected) : 비동기 작업이 실패한 상태
